# Conductor Development Setup Guide

## Prerequisites
- Node.js 18+ and npm
- Encore CLI installed (`curl -L https://encore.dev/install.sh | bash`)

## Quick Start (From Git Clone to Working App)

### 1. Clone and Install Dependencies

```bash
# Clone the repository
git clone [your-repo-url] conductor
cd conductor

# Install backend dependencies
cd backend
npm install

# Install frontend dependencies
cd ../frontend
npm install
cd ..
```

### 2. Set Up Backend Database

```bash
cd backend

# IMPORTANT: Set up the .env file with correct database URL
echo "DB_URL=postgresql://conductor-ee92:local@127.0.0.1:9500/ticket?sslmode=disable" >> .env

# Start Encore backend (this creates the database)
NODE_ENV=development encore run
```

**Keep this terminal running!** Open a new terminal for the next steps.

### 3. Initialize Database (New Terminal)

```bash
cd backend

# Reset the database (if needed)
encore db reset ticket

# Push the Prisma schema to create tables
DB_URL="postgresql://conductor-ee92:local@127.0.0.1:9500/ticket?sslmode=disable" \
  npx prisma db push --schema=ticket/schema.prisma --accept-data-loss

# Seed the database with sample data
NODE_ENV=development \
  curl -X POST -H "Authorization: Bearer local" http://localhost:4000/seed

# You should see: {"message":"Database seeded successfully"}
```

### 4. Verify Backend is Working

```bash
# Test the API
curl -H "Authorization: Bearer local" http://localhost:4000/dashboard/metrics

# You should see metrics JSON data
```

### 5. Start Prisma Studio

```bash
cd backend

# Start Prisma Studio to view your data
npx prisma studio --schema=ticket/schema.prisma

# Opens at http://localhost:5555
```

### 6. Start Frontend (Optional)

```bash
cd frontend
npm run dev

# Opens at http://localhost:3000
```

## Common Issues and Solutions

### Issue: "The table `public.users` does not exist"
**Solution:** The database URL is wrong. Make sure `.env` has:
```bash
DB_URL=postgresql://conductor-ee92:local@127.0.0.1:9500/ticket?sslmode=disable
```
Note: It's `ticket` not `conductor-db`!

### Issue: Prisma Studio shows no data
**Solution:** 
1. Stop Prisma Studio (Ctrl+C)
2. Fix the `.env` file (see above)
3. Restart Prisma Studio

### Issue: "Database server was reached but timed out"
**Solution:** Reset the database:
```bash
encore db reset ticket
```

### Issue: Authentication errors
**Solution:** Make sure you're running with `NODE_ENV=development`:
```bash
NODE_ENV=development encore run
```

## What's Happening Under the Hood

1. **Encore** creates a PostgreSQL database named `ticket` on port 9500
2. **Prisma** connects to this database using the `DB_URL` environment variable
3. **Local Auth** is enabled when `NODE_ENV=development`, accepting "local" as the auth token
4. **Seeding** creates sample users, tickets, and comments

## Environment Variables Explained

```bash
# .env file in backend/
DB_URL=postgresql://conductor-ee92:local@127.0.0.1:9500/ticket?sslmode=disable
```

- `conductor-ee92`: Database user (auto-generated by Encore)
- `local`: Password
- `127.0.0.1:9500`: Encore's PostgreSQL instance
- `ticket`: Database name (MUST match Encore service name)

## Testing Authentication

All backend API calls need authentication. In development, use:
```bash
curl -H "Authorization: Bearer local" http://localhost:4000/[endpoint]
```

## Available Endpoints

- `POST /seed` - Seed database with sample data
- `GET /dashboard/metrics` - Get dashboard metrics
- `GET /tickets` - List tickets
- `POST /tickets` - Create ticket
- More endpoints in `backend/*/encore.service.ts` files

## Data Model

The database has three main tables:
- **users** - System users (agents, staff, admins)
- **tickets** - Support tickets
- **comments** - Comments on tickets

See `backend/ticket/schema.prisma` for the full schema.

## Tips for New Developers

1. **Always run Encore with NODE_ENV=development** for local auth to work
2. **The database name is "ticket"** not "conductor-db" 
3. **Keep Encore running** in one terminal while working
4. **Use Prisma Studio** to inspect and modify data visually
5. **Check the .env file** if database connections fail
